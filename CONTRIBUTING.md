# Contributing

Thanks for helping improve this Big Data learning repository.

## What to Contribute

- Fix factual errors, typos, and broken links.
- Improve clarity of explanations.
- Add practical examples aligned with course topics.
- Improve scripts and setup instructions for macOS/Linux.

## Contribution Guidelines

- Keep changes focused and small.
- Preserve existing folder structure and naming style.
- Use clear headings and short sections in Markdown.
- Prefer simple, tested commands in scripts.
- Avoid adding unrelated content or marketing material.

## Content Quality Checklist

Before submitting:

- Verify technical accuracy.
- Confirm commands run as documented.
- Check links and file paths.
- Keep tone student-friendly and concise.

## How to Propose Changes

1. Fork the repository.
2. Create a branch with a clear name.
3. Make your edits.
4. Open a pull request with:
   - What changed
   - Why it changed
   - Any verification done

## Scope Notes

- `UNITS/` contains academic-focused notes and summaries.
- `Big Data ( Full Course)/` contains practical and setup-oriented content.
- Keep additions aligned with the existing syllabus/course flow.

## Reporting Issues

Please open an issue for:

- Wrong commands or outdated setup steps
- Conceptual mistakes
- Formatting or navigation problems

Thank you for improving this guide for future learners.
